# Reparto para 3 expositores

## Expositor 1 - Problema, arquitectura y pipeline

Responsable de explicar:

- Problema de datos dispersos en ciberseguridad.
- Objetivo de PEPA.
- Arquitectura general.
- Tablas T1, T2 y T3.
- Pipeline Spark.
- Join triple.
- Salida Parquet.

Frase clave:

> PEPA convierte datos dispersos en una base analitica lista para dashboard, ML y monitoreo.

## Expositor 2 - Filtros, Dashboard 01 y Dashboard 02

Responsable de explicar:

- Filtros globales.
- KPIs superiores.
- Dashboard 01: tendencia y distribucion.
- Dashboard 02: geografia y finanzas.
- Valor ejecutivo y financiero.

Frase clave:

> Estos dashboards permiten entender rapidamente volumen, impacto economico y paises o ataques prioritarios.

## Expositor 3 - Dashboard 03, Dashboard 04, limitaciones y futuro

Responsable de explicar:

- Dashboard 03: industria, severidad y datos comprometidos.
- Dashboard 04: ML y feed en vivo.
- Modo demo, laboratorio y produccion.
- Limitaciones actuales.
- Mejoras futuras.

Frase clave:

> PEPA esta listo para laboratorio y preparado para evolucionar hacia monitoreo real con logs de Suricata, Wazuh, Zeek o SIEM.

## Recomendacion de tiempo

Para una exposicion de 15 minutos:

```text
Expositor 1: 5 min
Expositor 2: 5 min
Expositor 3: 5 min
```

Para una exposicion de 20 minutos:

```text
Expositor 1: 6 min
Expositor 2: 7 min
Expositor 3: 7 min
```
